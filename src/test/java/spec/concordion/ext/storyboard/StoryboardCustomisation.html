Customisation

The Storyboard can be customised in several different ways:

  • addCardImage(): Add custom card image that will be shared by all tests
    • Must be called before test starts otherwise this will do nothing
    • It is possible to override stock card images if you use the same name
  • setAddCardOnThrowable(): Sets whether a card will be added to the storyboard when an uncaught exception occurs in the test
    • Defaults to true
  • setAddCardOnFailure(): Sets whether a card will be added to the storyboard when a failure occurs in the test
    • Defaults to true
  • setScreenshotTaker(): Set a custom screenshot taker
    • By default this is set to RobotScreenshotTaker which will capture the full visible screen
    • For a non gui application this won't be the most appropriate behaviour and you will want to set the screenshot taker to null
    • See the demo application for an example of a custom SeleniumScreenshotTaker.
  • setTakeScreenshotOnCompletion(): When the test completes and the ScreenshotTaker is not null the storyboard will take a screenshot of the current screen
    • Defaults to true
  • setAcceptsScreenshots(): Set to false to stop screenshot cards being added, this is useful in situations where might be looping and refreshing screen constantly and you don't want to show many screenshots of same screen in storyboard
